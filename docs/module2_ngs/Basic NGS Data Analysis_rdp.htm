Basic NGS Data Analysis

 

Module Introduction

 

In this module, you'll learn the basics of analyzing Next-Generation Sequencing (NGS) data. We'll start by looking at different NGS technologies and understanding the data they produce. Then, we'll examine how to spot Single Nucleotide Polymorphisms (SNPs) in resequencing data. We'll also cover how to make sure our data is good quality, filter out any errors, and do some basic number-crunching to understand what our SNP data is telling us.

Section 1. Variant calling

Learning Objectives:

  1. To develop proficiency in utilizing the Genome Analysis Toolkit (GATK) workflow for variant calling
  2. To understand the process of preparing raw data to meet the specifications of the GATK workflow
  3. Apply knowledge gained from GATK variant calling workflow to effectively identify and characterize genetic variants, including single nucleotide polymorphisms (SNPs) and small insertions/deletions (indels)

                    ○        http://samtools.sourceforge.net

 

Genome Analysis Toolkit (GATK)

GATK is a software package to analyze high-throughput sequencing data, developed by the Data Science and Data Engineering group at the Broad Institute. The current version (GATK4) includes more complex analysis such as copy number (CNV), structural variant (SV) and somatic variants. The current version of GATK also includes several utility functions for processing alignment files, VCF files and other complex processing workflows.

 

 

Input Files

  1. Raw reads
    1. Format: FASTQ (.fq, .fastq, .fastq.gz)
    2. FASTQ: Used to store both the nucleotide sequence data and quality scores
      1. 1st line:  Sequence identifier
      2. 2nd line: Sequence
      3. 3rd line: Quality sequence header  (usually a '+' character)
      4. 4th line: Quality scores corresponding to each base in the sequence

                        

 

  1. Reference
    1. Format: FASTA (.fasta, .fa, .fsa)
    2. FASTA: A text-based format for representing collections of nucleotide or protein sequences. The file consists of several blocks, each containing a header line and sequence lines. Typically, each block contains a sequence of a whole chromosome or a contig.
      1. Line 1: A header line starting with a '>' character, which contains information about the sequence. In the variant calling pipeline this is where the sequence identifier will be taken from.
      2. Line 2: Sequence (can occupy several lines)

                             

Intermediate Files

  1. Sequence Alignment/Map Format (SAM)
    1. Format for storing large nucleotide sequence alignments
    2. A TAB-delimited text format consisting of a header section, which is optional, and an alignment section
    3. Specification document: https://samtools.github.io/hts-specs/SAMv1.pdf

                  

  1. Binary SAM files (BAM)
    1. Binary version of the SAM format
    2. To view the file use: $ samtools view sample.sorted.bam
    3. Manual: https://samtools.github.io/hts-specs/SAMv1.pdf (Section 4)

 

 

  1. BAM Index files
    1. Samtools index – indexes SAM/BAM/CRAM files
    2. Syntax: samtools index [options] BAMfile

Output File

  1. Variant Call Format
    1. Store the most prevalent types of sequence variation, such as SNPs and small indels (insertions/deletions)
    2. Manual: https://samtools.github.io/hts-specs/VCFv4.2.pdf

Subsection 2: Variant Calling Pipeline

Description

This is a pipeline for variant calling based on best practices for GATK4

 

Requirements and Preparation

  1. Ubuntu installation on your laptop/workstation
  2. Installation of the necessary tools
    1. BWA, FastQC, FastP, GATK4, Picard, Samtools
  3. Demo dataset
    1. Reference: Ref.fasta
    2. Data: sample-1_1.fq.gz  sample-1_2.fq.gz

 

 

Workflow

         

 

  1.  Preparation and data import

tar -xzvf sample.tar.gz

 

  1. Create output directory

cd sample

mkdir Output

 

  1. Indexing reference
    1. Why: Indexing improves the ability of tools to rapidly access random locations of the file. It allows efficient retrieval of sequence data from the FASTA file.

bwa index Ref/Ref.fasta

    

samtools faidx Ref/Ref.fasta

           

java -jar $PICARD CreateSequenceDictionary -REFERENCE Ref/Ref.fasta -OUTPUT Ref/Ref.dict

 

  1. Perform quality control checks on the raw fastq data
    1. Why: to ensure that subsequent analyses are based on reliable data
    2. Tool: fastqc
      1. Functionality: perform quality control checks on raw sequencing data
      2. Syntax: fastqc <read1> <read2> <.. seqfileN>

fastqc Data/sample-1_1.fq.gz Data/sample-1_2.fq.gz
      1. Output: FastQC Report - identify potential issues such as adapter contamination, overrepresented sequences, and base quality distribution.
      2. Sample result:

                                     

      1. Visit https://www.bioinformatics.babraham.ac.uk/projects/fastqc/Help/ for more details
  1. Cleaning of reads
    1. Tool: Fastp
      1. Functionality: a fast all-in-one preprocessing tool for removing adapters, filtering, and trimming raw sequencing data
      2. Syntax: fastp -i <read1> -I <read2> -o <read1-cleaned> -O <read2-cleaned>

fastp -i Data/sample-1_1.fq.gz -I Data/sample-1_2.fq.gz -o

Data/sample-1_1-cleaned.fq.gz -O Data/sample-1_2-cleaned.fq.gz

 

      1. Output: Cleaned reads - high-quality, trimmed reads free of adapters and low-quality bases, ready for alignment

 

 

 

 

  1. Alignment
    1. Why: To determine the genomic locations of the reads, which is fundamental for downstream analyses such as variant calling, gene expression analysis, and structural variation detection
    2. Tool: bwa mem
      1. Functionality: Align sequencing reads to a reference genome to determine their origin and location
      2. Syntax: bwa mem <arguments> <path_ref.fasta> <path_read1.fq.gz> <path_read2.fq.gz> > <path_output.sam>

 

     Arg:

 -M            mark shorter split hits as secondary

 -t INT        number of threads

Type man bwa mem (to view manual page)

bwa mem -M -t 8 Ref/Ref.fasta Data/sample-1_1-cleaned.fq.gz Data/sample-1_2-cleaned.fq.gz > Output/sample.sam

 

      1. Output: Sequence alignment data / alignment output (SAM format).

 

 

  1. Sorting
    1. Why: 1) enable faster access to specific genomic regions and improve the performance of subsequent tools, 2) To save disk space; faster manipulation of files (faster to read)
    2. Tool: SortSam
      1. Functionality: Sorts a SAM or BAM file
      2. Syntax: java -jar $GATK SortSam --INPUT input.bam --OUTPUT file.sorted.bam -SORT_ORDER coordinate

                Arg:

--INPUT,-I <File>             Input BAM or SAM file to sort.  Required

--OUTPUT,-O <File>            Sorted BAM or SAM output file.  Required

--SORT_ORDER,-SO <SortOrder>  Sort order of output file.

coordinate           Sorts primarily according to the SEQ and POS fields of

the record

duplicate            Sorts the reads so that duplicates reads are adjacent

queryname            Sorts according to the readname

 

java -jar $GATK SortSam --INPUT Output/sample.sam --OUTPUT Output/sample.sorted.bam -SORT_ORDER coordinate

 

      1. Output: Sorted aligned reads, which are essential for downstream processing steps and variant calling (.sorted.bam file)

 

 

 

 

 

 

 

 

 

 

 

 

  1. Fixing pairs
    1. Why: To ensure the accuracy and reliability of subsequent analyses by correctly aligning and orienting reads relative to each other
    2. Tool: FixMateInformation
      1. Functionality: Ensures that all mate-pair information is in sync between each read and its mate pair
      2. Syntax: java -jar $GATK FixMateInformation --INPUT file.sorted.bam --OUTPUT file.fxmt.bam --CREATE_INDEX TRUE --VALIDATION_STRINGENCY LENIENT

     Arg:

--INPUT,-I <File>             The input files to check and fix. Required

--OUTPUT,-O <File>            The output file to write to.  Required

--CREATE_INDEX       <Boolean>     Whether to create an index when writing VCF or

coordinate Sorted BAM output. Default value: false. Possible values:  {true, false}

--VALIDATION_STRINGENCY <ValidationStringency>        

Validation stringency for all SAM files read by this program. Default value: STRICT. Possible values: {STRICT, LENIENT, SILENT}

STRICT               Do the right thing, throw an exception if something

looks wrong.

LENIENT                    Emit warnings but keep going if possible.

SILENT               Like LENIENT, only don't emit warning messages.

    

java -jar $GATK  FixMateInformation --INPUT Output/sample.sorted.bam --OUTPUT Output/sample.fxmt.bam --CREATE_INDEX TRUE --VALIDATION_STRINGENCY LENIENT
      1. Output: Correctly paired reads with accurate mate information (.fxmt.bam file)
  1. Add read groups
    1. Why: This tool is needed by subsequent GATK tools to ensure accurate interpretation and comparison of sequencing data. To read more about "read groups", please see this link
    2. Tool: AddOrReplaceReadGroups
      1. Functionality: Add or replace read group information in the alignment file (provides essential metadata about the sequencing experiment)
      2. Syntax: java -jar $GATK AddOrReplaceReadGroups --INPUT file.fxmt.bam --OUTPUT file.addrep.bam --RGID run100 --RGPU unit1 --RGLB lib1 -PL Illumina -SM sample -CN CN --VALIDATION_STRINGENCY LENIENT --CREATE_INDEX TRUE

                Arg:

--INPUT,-I <String>           Input file (BAM or SAM or a GA4GH url).  Required.

--OUTPUT,-O <File>            Output file (BAM or SAM).  Required.

--RGLB,-LB <String>           Read-Group library  Required.

--RGPL,-PL <String>           Read-Group platform (e.g. ILLUMINA, SOLID)  Required.

--RGPU,-PU <String>           Read-Group platform unit (eg. run barcode)  Required.

--RGSM,-SM <String>           Read-Group sample name  Required.

--RGCN,-CN <String>           Read-Group sequencing center name  Default value: null.

--RGID,-ID <String>           Read-Group ID  Default value: 1.

               

java -jar $GATK AddOrReplaceReadGroups --INPUT Output/sample.fxmt.bam --OUTPUT Output/sample.addrep.bam --RGID run100 --RGPU unit1 --RGLB lib1 -PL Illumina -SM sample -CN CN --VALIDATION_STRINGENCY LENIENT --CREATE_INDEX TRUE

      1. Output: Aligned reads with updated read group information (.addrep.bam)

 

 

 

 

 

  1. Mark duplicate reads
    1. Why: Some of the reads in FASTQ files may be duplicates, created during library preparation or during sequencing. Usually this represents only a small amount of reads Still, to avoid bias and inaccuracies in calling variants it is necessary to run this tool that marks the reads that are likely duplicates.
    2. Tool: MarkDuplicates
      1. Functionality: Identify and mark duplicate reads
      2. Syntax: java -jar $GATK MarkDuplicates --INPUT file.addrep.bam --OUTPUT file.mkdup.bam --CREATE_INDEX TRUE --VALIDATION_STRINGENCY LENIENT --METRICS_FILE Output/sample.mkdup.metrics --MAX_FILE_HANDLES_FOR_READ_ENDS_MAP 1000

Arg:

--INPUT,-I <File>             One or more input SAM or BAM files to analyze. Must be

coordinate sorted.

--OUTPUT,-O <File>            The output file to write marked records to.  Required.

--METRICS_FILE,-M <File>      File to write duplication metrics to. Required.

--MAX_FILE_HANDLES_FOR_READ_ENDS_MAP,-MAX_FILE_HANDLES <Integer>    

                                                Maximum number of file handles to keep open when

spilling read ends to disk.

java -jar $GATK MarkDuplicates --INPUT Output/sample.fxmt.bam --OUTPUT Output/sample.mkdup.bam --VALIDATION_STRINGENCY LENIENT --CREATE_INDEX TRUE --METRICS_FILE Output/sample.mkdup.metrics --MAX_FILE_HANDLES_FOR_READ_ENDS_MAP 1000

 

      1. Output: A duplicate-marked alignment file (.mkdup.bam) and a duplication metrics file (.metrics).

 

  1. Variant Calling
    1. Tool: HaplotypeCaller
      1. Functionality: Call SNPs and indels
      2. Syntax: java -jar $GATK HaplotypeCaller -R Ref.fasta -I file.mkdup.bam -O file.g.vcf -ERC GVCF --bam-output file.bamout.bam --showHidden true

                Arg:

--input,-I <GATKPath>         BAM/SAM/CRAM file containing reads  This argument must

Be specified at least once.Required.

--output,-O <GATKPath>        File to which variants should be written  Required.

--reference,-R <GATKPath>     Reference sequence file  Required.

--emit-ref-confidence,-ERC <ReferenceConfidenceMode>

                                         Mode for emitting reference confidence scores. Default

value: NONE. Possible values: {NONE, BP_RESOLUTION,GVCF}

--showHidden,-showHidden <Boolean>

                                                       Display hidden arguments  Default value: false. Possible

values:{true, false}

java -jar $GATK HaplotypeCaller \

    -R Ref/Ref.fasta \

    -I Output/sample.mkdup.bam \

    -O Output/sample.g.vcf \

    -ERC GVCF \

    --bam-output Output/sample.bamout.bam  \

    --showHidden true

 

      1. Output: A set of raw variant calls, including SNPs and indels (gVCF file)

Subsection 3. BCFtools query command

The main element in the structure of the command is the query string (or format string) that specifies what information you want to get. For each record in the VCF the query string will be evaluated and added as a row in the output

 

bcftools query

bcftools query -f '%CHROM\t%POS\t%REF\t%ALT[\t%SAMPLE=%GT]\n' file.vcf.gz

 

Extract variant QC metrics (bcftools query)

To get various QC metrics, include the names of the metrics with the % sign prefix into the query string.

There are some metrics, like read depth DP,  that are applicable to both variant as a whole or to each sample. To distinguish between these, use prefixes INFO and FORMAT

INFO/DP: depth across all samples, is placed into the INFO column

FORMAT/DP: depth at each individual sample genotype call, is mentioned in the FORMAT column, the actual information is located in the genotype matrix.

 

E.g. some variant level QC metrics

bcftools query -f '%CHROM\t%POS\t%REF\t%ALT\t%QUAL\t%QD\t%INFO/DP\n' file.vcf.gz > variant-DP.txt

 

( For comparison, here is an example of read depths of each sample:

bcftools query -f '%CHROM\t%POS\t%REF\t%ALT[\t%SAMPLE=%FORMAT/DP]\n' file.vcf.gz

)

Extract variant genotype matrix as a table (bcftools query)

Let us create a file by first putting row names, then dumping all the genotype calls:

 

bcftools query -f 'CHROM\tPOS\tREF\tALT[\t%SAMPLE]\n' file.vcf.gz | head -n 1 > out.txt

 

bcftools query -f '%CHROM\t%POS\t%REF\t%ALT[\t%GT]\n' file.vcf.gz >> out.txt

 

Note in the first command we don't use % except in the %SAMPLE - thus, only sample names will be interpolated into the string.

Subsection 5. Understanding alignment metadata

 

A lot of information about alignment of a read pair  to reference is contained in the "flags" column.

Each "flag" is a boolean (true or false) value encoded in a bit of the flags byte.

See https://en.wikipedia.org/wiki/SAM_(file_format)#Bitwise_flags

 

To easily retrieve the flags pertaining to a certain read alignment, e.g.

the first read in the input BAM file is

 

```

FCD0R2NACXX:7:2105:6385:19577#CCACATTC_24       163     Chr7    272768  0       83M     =       273143  458     AGTTGTTTCCTCCCGCCGTGGCCCACGCTTTATCCGCACTGCGCGGAAGCGTGTTGTTGGTTCTTCCACCGTGTTCCGTTTCC  BFHHHHJJIJIJJJIJJJGHJJJIIIIIJJJIIIIJIJJJJGHEDD@BDDDB@BDDDDDDDDDDDDCCDDDDDDDDDDDDDCD  NM:i:1  MD:Z:5C77       MC:Z:83M        AS:i:78 XS:i:78

```

Its flag column shows value 163.

Run the following:

samtools flags 163

 

The result:

0xa3  163   PAIRED,PROPER_PAIR,MREVERSE,READ2

To understand meanings of these abbreviations, start with running samtools flags without arguments

 

samtools flags

a combination of the following numeric flag values, or a comma-separated string

 0x800  2048  SUPPLEMENTARY  supplementary alignment

Module Summary

 

“Understanding the genetic basis of diversity through the discovery of Single Nucleotide Polymorphisms (SNPs) via variant calling is a fundamental step in gene discovery. By identifying and analyzing SNPs, researchers can uncover genetic variations that contribute to differences in traits. This knowledge is critical for pinpointing specific genes associated with those traits, facilitating advancements in fields such as genetics, breeding, and biotechnology.”

 

In this module, you have gained an understanding of the Variant Calling workflow. You have explored the essential tools and software used for variant discovery, including FASTQC, BWA, Picard, SamTools, and GATK.
