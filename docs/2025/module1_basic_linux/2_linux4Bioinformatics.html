Linux for Bioinformatics

Why Linux?

Linux is reliable, stable, and very powerful, coming with a complete development environment that includes compilers, toolkits, and scripting languages; it allows multi-user access, offers true multi-tasking, utilizes memory, CPU (with built-in support for multi-processors), and other hardware to the fullest; it's easily upgradable and free.

The Linux Shell

The Linux shell, rooted in the Unix shell, serves as a command-line interpreter (CLI) that enables users to interact with the operating system by entering text-based commands or executing scripts composed of multiple commands.

Users typically interact with the shell through a terminal emulator, which provides a user-friendly interface for command input and output display.

Common examples of Unix-like shells include:

  • Bourne Shell (sh): Developed by Stephen Bourne at AT&T Bell Labs, it's the original Unix shell. It's known for its simplicity and scripting capabilities, though it lacks features like command history and interactive use enhancements.
  • Bourne Again Shell (bash): A widely used shell that extends the Bourne Shell with features from the Korn Shell and C Shell, such as command history, job control, and improved scripting syntax.
  • C Shell (csh): Created by Bill Joy at the University of California, Berkeley, it introduced features like command history and aliases, with a syntax resembling the C programming language.
  • Korn Shell (ksh): Developed by David Korn at AT&T Bell Labs, it combines features from both the Bourne and C Shells, offering advanced scripting capabilities and improved performance.

Terminal Basics

In Linux:

You can typically open a terminal by searching for "terminal" in the application launcher or by using a keyboard shortcut like Ctrl + Alt + T.

In Windows:

To open the terminal for Windows Subsystem for Linux (WSL), you have a few options depending on your setup:

  • Using the Start Menu: Press the Windows key on your keyboard to open the Start Menu. Type "WSL" or "Linux" in the search bar. You should see options like "Ubuntu" or whichever Linux distribution you have installed under WSL. Click on it to open a terminal window directly.
  • Using Command Prompt or PowerShell: Open Command Prompt or PowerShell. Type wsl and press Enter. This will open a terminal window directly within your default Linux distribution under WSL.
  • Using Windows Terminal (if installed): If you have Windows Terminal installed, you can open it by searching for "Windows Terminal" in the Start Menu or by typing wt in the Run dialog (Win + R) and pressing Enter. Once Windows Terminal is open, you can select your Linux distribution from the dropdown menu or create a new tab for it.

In Mac:

You can open the terminal in several ways:

  • Using Spotlight Search: Press Cmd + Space to open Spotlight Search. Type "Terminal" and press Enter to open the Terminal application.
  • Using Finder: Open Finder, go to Applications, locate and open the Utilities folder, then double-click on the Terminal application to open it.
  • Using Launchpad: Click on the Launchpad icon in the Dock (it looks like a rocket). Find and click on the Terminal icon in the Launchpad grid to open it.
  • Using Spotlight in the Menu Bar: Click on the magnifying glass icon in the menu bar (top-right corner), type "Terminal," and press Enter to open the Terminal application.

Important Notes:

  • Linux/Unix is case-sensitive: Commands are case-sensitive, meaning WHO is different from who.
  • Shell behavior: The shell interprets the command you enter on the keyboard and executes them.
  • Command Examples: Commands are typically short and cryptic, like ls, vi, or mv.
  • Command Modifiers: Commands may also have modifiers for advanced options. For example, ls -l and mv -R are different from ls or mv, respectively.
  • Command Prompt: The $ symbol is the command prompt. Always type your command after the $.

Anatomy of the Linux Command

Consider the following command:

$ ls -l -h -S /tmp

This command can be broken down into its components:

  • ls — The command or utility that lists directory contents.
  • -l, -h, -S — Options or flags that modify the behavior of the command:
    • -l — Lists files in long format, showing detailed information such as permissions, owner, size, and modification date.
    • -h — Makes file sizes human-readable (e.g., 1K, 234M, 2G).
    • -S — Sorts files by size, with the largest files first.
  • /tmp — The argument specifying the directory to list; in this case, the temporary directory.

Note: The command can also be written without the hyphens in the options, as:

$ ls -lhS /tmp

Some commands accept options with or without the leading hyphen. For example, ls -lhS and ls -l -h -S are equivalent.

Some Basic Linux Commands

  • man — Displays the manual pages for other commands, providing detailed information on their usage and options.
  • info — Displays the command information pages, offering a more detailed and structured documentation format compared to man.
  • cal — Prints the calendar for the current month, allowing users to view dates and days quickly.
  • whoami — Prints the name of the current user, indicating the effective username of the user running the command. This command is part of the GNU coreutils project and is available on most Unix-like operating systems, including Linux and Windows.
  • who — Prints the names of the users who are currently logged into the computer, along with their login times and terminal information.

  • echo — Prints a text string on the screen, commonly used in scripts to display messages or output values.
